# KV Cache Affinity 代码走读清单

按**真实压缩请求流程**排序，从压缩触发到老化落地，一条线读下去。
每项给 `文件:函数` + 一句"看什么"。带 `vllm/` 前缀的是 vanilla vLLM 基线（vllm-0.18.0）。

> 两个最关键的点：**第 14**（token→block 换算）、**第 21**（正序选块 + 逆序入队）。
> 建议读法：先按 1→23 主线走一遍，最后回头看第六部分 vanilla 基线对照。

## 附：写入路径（Leg 0，释放的前提，可选读）

释放路径依赖"正常请求已把 block 打上 session 标记"。如需理解这个前提：
- `affinity_callback.py:async_log_success_event`(219) — 网关存 session_store
- `affinity_callback.py:170-171` — 注入 cache_salt/cache_sharing
- `v1/core/input_processor.py:register_input_processor` — salt 注入 prompt → 影响首块 hash
- `v1/core/kv_cache_manager.py` `allocate_slots` — 分配时登记 session
- `v1/core/kv_cache_session_manager.py` — `block_to_sessions` 维护
