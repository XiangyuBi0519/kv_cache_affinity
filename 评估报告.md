🔴 严重问题 (Critical Issues)
2.1 链表遍历中的严重 BUG (High Risk)
位置: kv_cache_affinity/v1/core/two_phase_block_queue.py 第 101-140 行 (popleft_n 方法)
问题描述:
在 popleft_n 方法的链表遍历中，内联修改链表指针会导致跳过块。
问题代码:
for _ in range(n):
    if curr_block is None:
        raise RuntimeError("Unexpected None block while popping from free list")
    ret.append(curr_block)
    last_block = curr_block
    curr_block = curr_block.next_free_block  # 获取下一个块
    # 问题：这里重置 last_block.next_free_block 会影响 curr_block！
    last_block.prev_free_block = None
    last_block.next_free_block = None
分析:
示例：当 n=2 时，假设列表为 [A, B, C]
第一次迭代:
- curr_block = A, last_block = A, 获取 curr_block.next_free_block = B
- 设置 last_block.next_free_block = None（即 A.next_free_block = None）
第二次迭代:
- curr_block 应该是 B，但我们在第一次迭代中已经将 A.next_free_block 设置为 None
- 在内联修改之前，curr_block = curr_block.next_free_block 能够正确获取 B
- 但是，这是我们获取到的最后一个有效的 next_free_block
实际影响:
- 当 n=3 时，只有前两个块会断开连接，第三个块可能仍然保持连接
- 这会导致部分块没有被正确重置，造成潜在的内存泄漏或不一致状态
修复方案:
def popleft_n(self, n: int) -> list[KVCacheBlock]:
    if n == 0:
        return []
    if self.num_free_blocks < n:
        raise ValueError(
            f"Not enough free blocks: requested {n}, "
            f"available {self.num_free_blocks}"
        )
    self.num_free_blocks -= n

    curr_block = self.fake_free_list_head.next_free_block
    # Step 1: Collect all blocks first
    ret = []
    for _ in range(n):
        if curr_block is None:
            raise RuntimeError("Unexpected None block while popping from free list")
        ret.append(curr_block)
        curr_block = curr_block.next_free_block
    
    # Step 2: Reset pointers for all collected blocks
    for block in ret:
        block.prev_free_block = None
        block.next_free_block = None
    
    # Update aging pointer if needed
    if self.aging in ret:
        self.aging = None
    
    if curr_block is not None:
        # Connect the fake head to the new first block
        self.fake_free_list_head.next_free_block = curr_block
        curr_block.prev_free_block = self.fake_free_list_head
    return ret
